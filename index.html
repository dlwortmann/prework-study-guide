HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attributes define additional information about an element.
  • Metadata is data about the webpage the browser uses to serve the webpage to the users. It is not seen by the users but contains important information about accessability, search engines and performance.
  • Parent elements hold child elements.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • External CSS style sheets are the most common and preferred as it keeps CSS rules in a separate file and make changing designs easier.
  • CSS Box Model is used to better understand how HTML element looks to users.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git workflow is important process for developers.
  • git add -A: commits files in working branch
  • git commit -m "short description of changes made": commits to repository
  • git pull origin main: pulls in base branch
  • git push origin "branch-name"
  • git checkout main: updates local repo with new merged feature

JavaScript

  • JS adds interactiviy to a website using tools to provide functionality.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Operators are symbols that produce a result based on two variables.
  • Some examples include mathematical operations, assignment, strict equality and Does Not Equal.
  • Conditionals test if a statment is true using if...else statements.
  • Events are code structures that listen for activity in browser then run a code in response.
  • Functions are the building blocks of JS and use a set of instructions to tell a computer how to perform a task.
